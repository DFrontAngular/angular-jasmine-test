Bloque 6 · Configuración de Karma

Entendiendo cómo se ejecutan realmente nuestros tests.

¿Qué es Karma?

Karma es el test runner que ejecuta nuestros tests en un navegador real. Angular CLI lo configura automáticamente, pero entender su configuración nos permite controlar mejor el entorno de ejecución.

Estructura base del karma.conf.js

El archivo karma.conf.js define cómo se ejecutan los tests: frameworks, navegadores, reporters y comportamiento. 

{{ baseConfigCode }}

Frameworks

Define qué motor de testing utilizamos. En Angular normalmente usamos Jasmine.

{{ frameworksCode }}
@angular-devkit/build-angular integra Karma con el CLI.

Reporters

Los reporters determinan cómo se muestran los resultados de los tests.

{{ reportersCode }}

El reporter coverage genera métricas de cobertura de código.

Coverage

Permite generar informes HTML y resumen en consola. Es clave para medir calidad del testing. 

{{ coverageCode }}

Navegadores

Define dónde se ejecutan los tests. En CI se recomienda usar ChromeHeadless.

{{ browsersCode }}

Entorno Headless

Un entorno headless ejecuta el navegador sin interfaz gráfica. Es fundamental en entornos CI/CD donde no existe entorno visual.

ChromeHeadless es más rápido, más estable y reduce problemas relacionados con renderizado gráfico. 

{{ headlessCode }}
En CI siempre usar ChromeHeadless para evitar errores de entorno gráfico y mayor estabilidad.

Configuraciones útiles

Algunas configuraciones ayudan a evitar tests intermitentes y mejorar la estabilidad.

{{ usefulConfigCode }}
  • random: false → Evita orden aleatorio en tests.
  • timeoutInterval → Aumenta el tiempo máximo.
  • logLevel → Reduce ruido en consola.
  • clearContext: false → Mantiene resultados visibles en navegador.

Reporters LCOV y SonarQube

LCOV es el formato estándar que muchas herramientas de análisis estático utilizan para leer cobertura de código.

SonarQube solo interpreta cobertura en formato LCOV, por eso debemos habilitar el reporter correspondiente. 

{{ lcovCode }}
  • lcov → Genera archivo + integración visual
  • lcovonly → Solo genera archivo .info sin HTML
SonarQube busca el archivo lcov.info para calcular métricas.

SonarQube no ejecuta los tests. Solo analiza los archivos generados. El archivo lcov.info contiene:

  • Líneas cubiertas
  • Funciones cubiertas
  • Branches cubiertos
  • Porcentaje global

Ignorar archivos en cobertura

No todos los archivos deben contarse para cobertura. Podemos excluir archivos técnicos o de configuración. 

{{ coverageIgnoreCode }}
Excluir archivos debe hacerse con criterio, nunca para "maquillar" métricas.

Buenas prácticas

  • ✔ Usar ChromeHeadless en CI
  • ✔ Activar coverage en pipelines
  • ✔ No desactivar random sin motivo
  • ✔ Ajustar timeouts solo si es necesario
Karma controla el entorno de ejecución, pero la calidad depende del diseño de los tests.
