Bloque 3 · Primer test de componente

En Angular, la mayor parte del valor está en testear componentes.

Qué estamos probando

Un componente standalone con:

  • Un Input
  • Un Output
  • Interacción del usuario
  • Renderizado en el DOM

Qué NO estamos probando

  • Métodos privados
  • Implementación interna
  • Change detection manual
Si un test necesita conocer detalles internos del componente, probablemente esté mal planteado.

Idea clave

Un test de componente en Angular es un test de integración.

Probamos el comportamiento observable desde el punto de vista del usuario.

El componente que vamos a testear

Antes de escribir tests, es importante entender qué hace el componente y qué responsabilidad tiene.

TaskItemComponent es un componente de presentación:

  • Recibe datos a través de un Input
  • Renderiza información en el DOM
  • Emite eventos cuando el usuario interactúa
Este tipo de componente es ideal para aprender testing en Angular porque su comportamiento es fácil de observar.
